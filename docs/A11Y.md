# Audit d'accessibilité & contraste

Lisibilité visée : club en plein air (terrains ensoleillés), deux thèmes (clair / sombre).
Ratios WCAG calculés sur les paires de couleurs des deux palettes (`src/global.css`).

## Contraste (ratios WCAG)

| Paire | Clair | Sombre | Verdict |
|---|---|---|---|
| Texte principal `on-surface` / `background` | 16,3:1 | 14,3:1 | AA ✓ (et AAA) |
| Texte secondaire `on-surface-muted` / `background` | 5,6:1 | 7,6:1 | AA ✓ |
| `error` / `background` (clair) | 6,1:1 | — | AA ✓ |
| **Blanc / `primary` #ff5700** (boutons) | 3,17:1 | 3,17:1 | **AA grand texte seulement** |
| **Orange #ff5700 en texte / fond clair** | 3,01:1 | — | **AA grand texte seulement** |
| Orange #ff5700 en texte / fond sombre | — | 5,8:1 | AA ✓ |

## Décisions

- **Texte principal et secondaire** : conformes AA (souvent AAA) dans les deux thèmes — lisibles en extérieur.
- **Boutons blanc-sur-orange** : `#ff5700` est l'accent de marque imposé (BRIEF), avec texte blanc. Ratio 3,17:1 = conforme AA pour **grand texte** uniquement. Mitigation : libellés de bouton en gras et hauteur de cible ≥ 44 px ; l'orange reste réservé aux actions (CTA), pas au petit texte.
- **Orange en texte sur fond clair** (variante `ghost`, liens) : 3,01:1 → à réserver aux libellés d'action (taille bouton), **jamais pour du petit texte/légende** sur fond clair. Sur fond sombre, l'orange-texte passe AA.
- **Minuteur & compteurs** : grande taille Lexend (timer 48px) → lisibles à distance.
- **Cibles tactiles** : `Button` a une hauteur minimale de 44 px.

## Accessibilité de base

- **Libellés** : contrôles dont le texte visible est un symbole (`−`/`+` terrains, `←` ronde précédente) reçoivent un `accessibilityLabel` français explicite ; les cartes de créneau tappables annoncent heure, niveau et nombre d'inscrits. `Button`/`Chip` portent `accessibilityRole` (et `accessibilityState` pour la sélection).
- **Dynamic Type** : les tailles `text-[NNpx]` suivent le facteur d'échelle système par défaut (RN `allowFontScaling`) ; mises en page en flex tolérant l'agrandissement.
- **Lecteurs d'écran** : écrans construits sur composants RN standards (Text/Pressable/ScrollView) → parcourables avec VoiceOver / TalkBack ; les icônes d'onglets sont décoratives (le libellé d'onglet porte l'intitulé).

## À surveiller (itératif)

- Vérifier visuellement en plein soleil sur appareil réel (couvert par la QA manuelle, issue #60).
- Si un besoin de petit texte orange sur clair apparaît, introduire un token `primary-text` plus foncé.
